# SQL VS NoSQL


## SQL Databases
- Fixed Schema
- Strong Consistency
- ACID transactions
- Suitable for relational data

### Use Cases
- Blogs
- Financial systems
- User management


## NoSQL Databases
- Flexible Schema
- Eventual consistency
- Horizontally scalable
- High read/write throughput

### Use Cases
- Chat systems
- Logs


## Conclusion
- SQL prioritizes correctness and relationships
- NoSQL prioritizes scalability and performance
